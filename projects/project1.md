# Project 1: How-To Guide

## Overview

In this assignment, you will create a clear, user-friendly instructional guide that teaches someone how to complete a specific task.

This project introduces core principles of technical writing: clarity, audience awareness, organization, and usability. You will also begin using a professional documentation workflow with VS Code, Markdown, and GitHub.

Your final document should be something a real person could successfully follow without additional help.

## Learning Objectives

By completing this assignment, you will:

* Write clear, step-by-step instructions for a defined audience
* Organize information using headings, lists, and visual hierarchy
* Integrate visuals (screenshots, diagrams, or images) effectively
* Practice writing in Markdown
* Use revision and feedback to improve usability

## Assignment Description

Choose a task that:

* Can be completed in 10–30 minutes
* Requires at least 5 steps
* Can be explained to a beginner

You will write a guide that includes:

* A clear introduction explaining the task and its purpose
* A list of required materials, tools, or prerequisites
* Step-by-step instructions in logical order
* At least 2–3 visuals (screenshots, photos, or diagrams)
 * We will talk about adding visuals in-class
* A troubleshooting or FAQ section
* A total of at least 650 words

## Possible Topics

You may choose your own topic, or select from examples like:

* Using a specific app or tool
* Completing a practical task (e.g., formatting a resume in Word)
* A hobby or skill (as long as it can be taught clearly)
* Putting something together

All topics must be approved before you begin drafting.

## Required Components

Your final submission must include:

### 1. Document (Markdown File)

Your guide should include:

* Title
* Introduction (what the user will learn + who it’s for)
* Materials/Prerequisites
* Numbered steps
* Visuals with captions
* Troubleshooting / FAQ section
* Conclusion

### 2. Visuals

* Minimum of 2–3 relevant visuals
* Must support the instructions (not decorative)
* Include captions or labels explaining what the reader is seeing

### 3. GitHub Submission

* Your document must be uploaded to a GitHub repository
* Proper file naming and organization required
* Submitted to Canvas via repository link

## Formatting & Style Guidelines

* Use Markdown syntax correctly (`#`, `##`, lists, bold, etc.)
* Write in a concise, direct style
* Use numbered steps for instructions
* Avoid long paragraphs: use headings and bullet points where relevant
* Maintain consistent tone and formatting

## Audience Requirement

You must define a specific audience (e.g., "someone assembling furniture" or "an older person using an app for the first time").

Your instructions should match your audience’s:

* Knowledge level
* Technical familiarity
* Needs and expectations

## Evaluation Criteria

Your work will be evaluated based on:

* **Clarity & Usability** – Can a user successfully complete the task?
* **Organization & Structure** – Logical flow, effective formatting
* **Audience Awareness** – Appropriate level of detail and tone
* **Style & Mechanics** – Grammar, conciseness, readability
* **Technical Execution** – Proper use of Markdown and GitHub

## Tips for Success

* Test your instructions on someone else
* Be specific: avoid vague steps like “click the button”
* Think like a beginner
* Use visuals where confusion might occur
* Revise for clarity, not just correctness
